# Qwen3.5 模型 S3 推理使用说明

## 概述

通过 sbatch 在计算节点上从 S3 挂载并加载 Qwen 系列模型进行推理，无需将模型下载到本地存储，适合大模型场景（本地存储有限）。

## 使用方法

脚本路径：`/mnt/petrelfs/leihaodong/local_model/test_qwen_s3_sbatch.sh`

```bash
cd /mnt/petrelfs/leihaodong/local_model
sbatch test_qwen_s3_sbatch.sh <模型S3地址> <模型名>
```

### 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| 模型S3地址 | 模型在 S3 上的完整路径 | `s3://datafrontier/leihaodong/Qwen/Qwen3.5-0.8B/` |
| 模型名 | 模型名称（用于日志） | `Qwen3.5-0.8B` |

### 示例

```bash
# 使用 Qwen3.5-0.8B
sbatch test_qwen_s3_sbatch.sh s3://datafrontier/leihaodong/Qwen/Qwen3.5-0.8B/ Qwen3.5-0.8B

# 使用其他模型（替换地址和名称即可）
sbatch test_qwen_s3_sbatch.sh s3://datafrontier/leihaodong/Qwen/其他模型/ 其他模型
```

### 默认值

不传参数时，默认使用：
- 模型地址：`s3://datafrontier/leihaodong/Qwen/Qwen3.5-0.8B/`
- 模型名：`Qwen3.5-0.8B`

```bash
sbatch test_qwen_s3_sbatch.sh
```

## 输出

- **标准输出**：`/mnt/petrelfs/leihaodong/local_model/qwen_s3_test_<job_id>.out`
- **错误输出**：`/mnt/petrelfs/leihaodong/local_model/qwen_s3_test_<job_id>.err`

推理结果示例：
```
=== 输入: hi ===
=== 输出: Hello! How can I help you today? 😊
```

## 流程说明

1. **挂载**：在计算节点 NVMe 上挂载 S3 桶（s3mount）
2. **读取**：直接从挂载点加载模型，按需读取，利用 NVMe 缓存加速
3. **推理**：使用 conda `qwen3` 环境，加载模型并回答 "hi"
4. **卸载**：任务结束后自动卸载

## 依赖

- conda 环境 `qwen3`
- 模型需已上传至 S3，路径正确
- 需有 DataFrontier_Knowledge 分区权限
