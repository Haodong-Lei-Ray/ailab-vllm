# Qwen3.5 模型下载到 S3 使用说明

使用 s3mount + sbatch 在计算节点上将 HuggingFace 上的 Qwen3.5 模型下载到 S3 对象存储。

---

## 一、前置条件

- **HF_TOKEN**：在 `local_model/token.sh` 中配置，或设置环境变量
- **AK/SK**：在 `~/.bashrc` 或 `~/.aws/credentials` 中配置（用于 s3mount 访问 datafrontier 桶）
- **conda 环境**：脚本会在 conda qwen3 环境下运行

---

## 二、使用命令

```bash
cd /mnt/petrelfs/leihaodong/local_model

sbatch download_qwen_s3mount_filebyfile.sh <REPO_ID> <S3_PREFIX>
```

| 参数 | 说明 |
|------|------|
| `REPO_ID` | HuggingFace 模型 ID |
| `S3_PREFIX` | S3 桶内路径（对应 `s3://datafrontier/<S3_PREFIX>/`） |

---

## 三、示例

### Qwen3.5-0.8B

```bash
sbatch download_qwen_s3mount_filebyfile.sh Qwen/Qwen3.5-0.8B leihaodong/Qwen/Qwen3.5-0.8B
```

目标路径：`s3://datafrontier/leihaodong/Qwen/Qwen3.5-0.8B/`

### Qwen3.5-27B

```bash
sbatch download_qwen_s3mount_filebyfile.sh Qwen/Qwen3.5-27B leihaodong/Qwen/Qwen3.5-27B
```

### Qwen3-30B

```bash
sbatch download_qwen_s3mount_filebyfile.sh Qwen/Qwen3-30B-A3B-Instruct-2507 leihaodong/Qwen/Qwen3-30B-A3B-Instruct-2507
```

目标路径：`s3://datafrontier/leihaodong/Qwen/Qwen3.5-27B/`

### 其他模型

```bash
sbatch download_qwen_s3mount_filebyfile.sh Qwen/Qwen3.5-72B leihaodong/Qwen/Qwen3.5-72B
```

---

## 四、流程说明

1. **挂载 S3**：在计算节点 NVMe 上通过 s3mount 挂载 `datafrontier` 桶
2. **逐文件处理**：对每个模型文件：
   - 从 HuggingFace 下载到本地临时目录
   - 写入挂载点（即上传到 S3）
   - 删除本地文件
3. **空间占用**：每次只保留一个文件，适用于本地存储有限（如 150G）的场景

---

```bash
# 查看作业状态

tail -f /mnt/petrelfs/leihaodong/local_model/slurm-<job_id>.out
```

## 六、验证是否成功

```bash
aws s3 ls s3://datafrontier/leihaodong/Qwen/ --endpoint-url http://d-ceph-ssd-inside.pjlab.org.cn
```

---

## 七、注意事项

- **单文件大小**：若某个文件超过本地剩余空间（如 200G 的 safetensors），此脚本无法处理，需使用直接下载到挂载点的方式
- **执行环境**：必须在计算节点运行，管理机不支持 s3mount
- **分区**：默认使用 `DataFrontier_Knowledge`，可在脚本中修改 `#SBATCH -p`
